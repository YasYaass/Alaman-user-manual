Homepage / Introduction

Welcome to the ALAMAN User Manual. This document explains how to register, record runs, view routes and statistics, post achievements, participate in events, and — for admins — manage club events and registration forms. The ALAMAN app is built with Flutter and targets Android devices.

Purpose of this manual

This manual provides step-by-step instructions and placeholders for screenshots to help end users and administrators use the ALAMAN mobile app. Use this as a reference during onboarding, troubleshooting, and for creating tutorials.

Supported platform: Android (built with Flutter)

Full User Guide

Registering an account

Follow these steps to create a new account.

  1. Open the ALAMAN app on your Android device.
  2. Tap Create Account (or Register).
  3. Enter your full name, email address, and a secure password.
  4. Accept the terms and privacy policy, then tap Sign Up.
  5. Verify your email if prompted (check your inbox for a verification link).

Logging into the app

Steps to sign in to ALAMAN after registering.

  1. Open the ALAMAN app.
  2. Tap Log In.
  3. Enter your registered email and password.
  4. Tap Sign In. Optionally enable "Remember me" for quicker access.
  5. If you forget your password, tap Forgot password and follow the reset flow.

Recording a run using GPS

How to start and stop GPS-based tracking.

  1. From the home screen, tap Start Run or the GPS/Record icon.
  2. Allow location permissions when prompted (necessary for GPS tracking).
  3. Wait for GPS lock (the app will show an indicator when accurate).
  4. Tap Start to begin recording.
  5. During the run you can pause or end the run. Tap End when finished.
  6. Review the saved route, summary and stats, then tap Save or Discard.

Viewing completed routes and statistics

Access your run history and detailed stats.

  1. Open the app and go to My Runs or Routes.
  2. Select a completed run from the list to see the route map and summary.
  3. Review statistics such as distance, duration, pace, elevation, and calories.
  4. Use the export or share button to share the route or export GPX if available.

Posting achievements

Share achievements with your club.

  1. Open Achievements or Activity Feed.
  2. Tap New Post or Share Achievement.
  3. Attach a route, photo, or write a message describing your achievement.
  4. Choose visibility (club-only or public within the group).
  5. Tap Post to publish.

Registering for club events

Sign up for events announced by the club owner.

  1. Open Events in the app.
  2. Browse upcoming event announcements and select one to open details.
  3. Tap Register or Sign Up.
  4. Fill out any required registration fields (e.g., emergency contact, t-shirt size).
  5. Confirm your registration and check for a confirmation message or email.

Admin-only features

Features available to club owners / admins. Access requires an admin account.

Creating event announcements

  1. Log in with an admin account and open the Admin Panel or Events management area.
  2. Tap Create Event.
  3. Enter event title, date/time, location, description, and any images.
  4. Set registration options (capacity, registration deadline, fee if applicable).
  5. Publish the announcement so members can view and register.

Creating and managing registration forms

  1. In the Admin Panel, open Registration Forms or event-specific settings.
  2. Create a new form or edit an existing one. Add fields like emergency contact, medical notes, shirt size.
  3. Configure which fields are required and any privacy options.
  4. Save and attach the form to an event. Monitor registrations from the admin dashboard.

Glossary

Definitions of key terms used in the ALAMAN app and manual.

GPS Tracking
Use of the device's GPS sensor to record geographic coordinates over time in order to produce a route map of a run or activity.
Achievement
A milestone or accomplishment posted by a user (example: personal best distance, fastest pace, completed challenge) which can be shared within the club.
Event
A scheduled club activity (race, group run, meeting) announced by an admin; users can register to participate.
Admin Panel
The administrative interface available to club owners where events, registration forms, announcements and user management are performed.
Flutter
An open-source UI toolkit by Google used to build natively compiled applications for mobile (Android and iOS), web, and desktop from a single codebase. ALAMAN is developed with Flutter.
User Profile
A user's account page containing profile information, statistics, activity history, and settings.
Statistics
Quantitative summaries of runs and activities such as total distance, average pace, elevation gain, and number of activities.
Authentication
The process of verifying a user's identity, typically via email/password sign-in, social login, or token-based authentication used by the app to secure user accounts.

Documentation Access (GitHub Pages)

This user manual can be hosted as a static site using GitHub Pages. To publish:

  1. Create a GitHub repository named alaman-user-manual (or any name you choose).
  2. Upload this file as index.html to the repository root, or place it in a docs/ folder.
  3. In the repository Settings → Pages, set the Source to the branch and folder where the file is stored (e.g., main / root or main / docs), or deploy to gh-pages.
  4. After a short time, your manual will be available at the GitHub Pages URL shown in Settings.

Example URL format (replace USERNAME with your GitHub login):

https://Assel.github.io/Alaman-user-manual/

Notes:

  • If you put the file in a docs/ folder on main, choose that folder in Pages settings.
  • For a more advanced site with search, navigation and theming, consider MkDocs or Docusaurus and deploy via GitHub Actions (not required for this static file).
